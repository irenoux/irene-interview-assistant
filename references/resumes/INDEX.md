## How to use this library

**At every ATS step**: read all versions listed as "latest" for each track, compare against the JD, select the closest match, state which was chosen and why.

**Naming convention for customized versions**: `{track}_{version}_{company}_{YYYYMMDD}.txt`
Example: `platform_v2.3_zillow_20260302.txt`

Customized versions are stored in this same folder. When a customized version exists for a company, use it as the starting point for any subsequent work on that role (saves re-doing prior edits).
